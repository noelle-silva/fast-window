# 更新日志

本文件记录 Fast Window 宿主可感知的行为变更。所有条目均使用中文描述。

## 未发布

### 新增

- AI Once 应用设置新增模型服务等待秒数，默认等待调整为 600 秒，问答和模型列表刷新会按用户保存的等待时间执行。
- 宿主新增独立的能力登记簿记录，用户选取的应用能力会单独保存并显示在主页搜索列表中。

### 调整

- 拆分注册应用的宿主快捷命令与 App 能力 API：应用注册页只管理宿主快捷命令，能力登记簿只管理可调用能力，主页会用不同角标区分两类按钮。
- 应用注册页新增“读取宿主快捷命令”能力；当 App 未运行时会提示是否启动后读取，并将结果写回注册页内的宿主快捷命令列表。
- 统一宿主侧运行时声明通路：宿主快捷命令与 App 能力 API 分别读取，普通 App 动作成功响应不再夹带声明清单。
- 统一宿主侧能力返回文字协议：应用能力调用必须主动返回标准展示文字，宿主、Quick Bar 与主窗口能力入口不再识别旧式回答。
- 统一宿主侧能力配置选项整理：能力配置会提供整理后的标准清单，Quick Bar 与主窗口能力入口不再分别拆解原始返回。
- 优化主窗口唤醒快捷键行为：主窗口已出现但未聚焦时，首次按快捷键会优先聚焦窗口；只有主窗口已经处于焦点状态时，再次按快捷键才会隐藏窗口。
- 优化注册 App 窗口模式快捷键行为：窗口已运行但不在前台时，首次按快捷键会优先找回窗口；只有窗口已经在前台时，再次按快捷键才会隐藏窗口。
